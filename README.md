# micaelalafratta.com

Personal portfolio site for Micaela Lafratta — Communications Strategist & Data Governance professional.

Built with [Astro](https://astro.build/) · Deployed on [Netlify](https://www.netlify.com/) · AI-assisted development with [Claude](https://claude.ai/)

🌐 [www.micaelalafratta.com](https://www.micaelalafratta.com)

---

## About this project

This site was built starting from an Astro template, then progressively shaped through a collaborative workflow between my own editorial judgment and Claude (Anthropic's AI assistant) as a coding partner.

I have a front-end background through CMS platforms — WordPress, visual builders, design systems — but not through code. I know exactly what I want a site to look like and how it should be structured; I just don't write the CSS and JavaScript to get there myself.

This project is a demonstration of a skill I consider professionally relevant: **using AI tools with enough precision and criteria to produce real, production-quality output.** I defined the typography, colour palette, visual style, and site architecture. Claude helped me translate those decisions into code and adapt the Astro template to match them.

Every output was reviewed by me and only shipped when it matched my vision. That required being specific in how I briefed Claude, catching mistakes, pushing back when something was off, and understanding enough of the result to evaluate it — even without writing it from scratch.

---

## What I built

The site covers three content collections:

- **Work** — 12 project case studies across communications strategy, EU-funded projects, brand and digital work
- **Experience** — 4 role pages with scope, responsibilities, and context
- **Data Projects** — Data analysis and visualisation projects (Power BI, Python, SQL)

Each entry has its own dedicated page with structured content: project context, scope, responsibilities, and measurable impact.

## How AI was used

Claude was involved throughout the build — not as an autopilot, but as a technical collaborator I directed at every step.

**Specific contributions:**

- Generating and modifying Astro component code based on my design briefs
- Debugging layout and styling issues I identified visually
- Building the content schema for work, experience, and data project collections
- Writing and refining individual project pages from my raw notes and briefs
- Adapting the template's default structure to better reflect my professional positioning
- Suggesting and implementing responsive behaviour and dark mode adjustments

**What I contributed:**

- Typography, colour palette, visual style, and overall aesthetic direction
- Site architecture and content structure — what sections exist and how they relate
- All editorial decisions — what to include, how to frame it, what to leave out
- Design direction for each component: spacing, hierarchy, layout behaviour
- Quality control — reviewing every output against my design intent before it went live
- The underlying professional experience the site is built to communicate

## Running locally

```bash
# Install dependencies
npm install

# Start dev server
npm run dev

# Build for production
npm run build
```

Requires Node.js 18+.
